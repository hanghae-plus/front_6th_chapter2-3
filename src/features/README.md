# Features Layer

사용자가 수행하는 주요 기능을 담는 레이어입니다.

## 역할

- 비즈니스 로직이 포함된 기능 단위
- 여러 페이지에서 재사용되는 기능
- 특정 Entity와 연결된 상호작용

## 구조

```
features/
└── feature-name/
    ├── ui/         # 기능 UI 컴포넌트
    ├── model/      # 비즈니스 로직, 상태
    ├── api/        # 기능별 API 요청
    └── config/     # 기능 설정, Feature Flag
```

## 생성 기준

- **생성**: 여러 페이지에서 재사용되는 기능
- **제외**: 단일 페이지 전용 기능은 Page에 위치
- **예시**: 인증, 검색, 댓글 작성, 좋아요

> 💡 새로운 팀원이 Page와 Feature만 봐도 앱의 기능 구조를 파악할 수 있도록 구성하세요.
