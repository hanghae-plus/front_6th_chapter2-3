# Pages Layer

웹/앱의 화면 단위에 해당하는 레이어입니다.

## 역할

- 화면별 UI 구성
- 데이터 패칭 및 상태 관리
- 라우팅 연결점

## 구조

대부분 페이지 1개 = Slice 1개로 구성됩니다.

```
pages/
└── page-name/
    ├── ui/         # 페이지 UI 컴포넌트
    └── api/        # 페이지별 데이터 요청
```

## 규칙

- 재사용되지 않는 UI는 Page 내부에 위치
- 복잡한 비즈니스 로직은 Features로 분리 고려
- 간단한 상태만 컴포넌트 내부에서 관리
